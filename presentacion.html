Overview

Sistema médico integral que combina gestión de historiales clínicos con análisis automatizado de imágenes médicas mediante IA. Diseñado para facilitar la administración de pacientes, consultas, exámenes y diagnósticos asistidos por inteligencia artificial.

Features

User Management

  • Role-based access control
  • Admin, Doctor, Nurse roles
  • Secure authentication

Patient Management

  • Medical history tracking
  • Appointment management
  • Document storage

AI Integration

  • Image analysis
  • Automated diagnostics
  • Medical validation

Technologies

Backend

Installation

Prerequisites

  • Node.js (v20 or higher)
  • PostgreSQL (v14 or higher)
  • Docker & Docker Compose

Steps

# Clone repository git clone https://github.com/runinbk/medical-ai-system.git cd medical-ai-system # Setup environment cp .env.example .env # Deploy with Docker docker-compose -f docker-compose.prod.yml build docker-compose -f docker-compose.prod.yml up -d

API Endpoints

Authentication Endpoints

POST /api/auth/login { "email": "user@example.com", "password": "password123" } POST /api/auth/registro { "nombre": "User Name", "email": "user@example.com", "password": "password123" }

Protected Routes

  • GET /api/pacientes - List patients
  • GET /api/medicos - List doctors
  • GET /api/consultas - List appointments
  • GET /api/examenes - List medical tests
  • POST /api/diagnosticos-ia - AI diagnosis

Authentication

JWT Authentication

All protected endpoints require a valid JWT token in the Authorization header:

Authorization: Bearer your_jwt_token_here

Default Users

{ "admin": { "email": "admin@medical.com", "password": "admin123" }, "doctor": { "email": "doctor@medical.com", "password": "doctor123" } }
